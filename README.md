# Delta Handbook

Repositório central de conhecimento do Delta, contendo documentação técnica, arquitetura de software, UX research, banco de dados, engenharia de dados, operações e padrões de desenvolvimento.


## Objetivo

Centralizar informações importantes em um único local, promovendo:

- Compartilhamento de conhecimento;
- Padronização de processos e práticas;
- Documentação de decisões técnicas;


## 🗂️ Estrutura do Repositório

```text
delta-handbook/
│
├── dados/
├── devops/
├── eqs/
├── geral/
└── ux/
```

## 📐 Padrões de Documentação

Sempre que possível, os documentos devem:

- Possuir título e objetivo claros;
- Conter contexto e justificativas;
- Ser escritos em Markdown (`.md`);
- Utilizar diagramas quando necessário;
- Manter histórico de atualizações relevantes.


### ✒️ Nomeação

Utilizar nomes em minúsculo e separados por hífen:

```text
api-autenticacao.md
modelo-relacional.md
jornada-do-usuario.md
```


## 🤝 Contribuindo

Antes de criar ou alterar documentação:

1. Verifique se já existe conteúdo semelhante.
2. Mantenha o padrão definido neste repositório.
3. Documente decisões e justificativas.
4. Solicite revisão quando necessário.
